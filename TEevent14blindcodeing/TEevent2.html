Blind Coding Challenge

Blind Coding Challenge is a unique and challenging event where participants must write code without being able to see what they are typing. This tests their programming logic, syntax knowledge, and muscle memory.

Rules:

1. Individual Participation: Participants compete individually.

2. Time Limit: Each participant is given a set amount of time (e.g., 15-30 minutes) to complete the coding challenge.

3. Programming Language: Participants can choose ( C, Python, Java).

4. Blindfolding: The participantâ€™s screen will be turned off, or they will wear a blindfold, preventing them from seeing their code as they type.

5. Simple Problem: The problems should be simple but require correct logic, such as basic algorithms (sorting, loops, or mathematical operations).

6. Scoring Criteria: Points are awarded based on the following: o Code Accuracy: Whether the code runs without errors or crashes. o Code Completeness: How close the code is to solving the given problem. o Logic and Approach: Even if there are syntax errors, logical correctness is considered.

7. Limited Debugging: Participants may be given an additional 5 minutes after the challenge to correct any syntax or minor logical errors, but excessive changes will result in point deductions.
