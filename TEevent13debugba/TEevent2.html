Debugging Battle

Debugging Battle e is a competitive event where participants are given buggy code and must identify and fix errors as quickly as possible. The goal is to test problem-solving skills, attention to detail, and debugging proficiency.

Rules:

1. Individual or Team Participation: Participants can compete individually or in teams (2-3 members).

2. No Internet: Participants cannot use the internet to search for solutions but can use built-in IDE features (e.g., debuggers).

3. Time Limit: Each round has a set time limit (e.g., 20-30 minutes) to debug the code.

4. Levels of Difficulty: There will be multiple rounds with increasing difficulty. The initial rounds might have basic syntax errors, while later rounds will involve logical bugs or complex runtime issues.

5. Environment: A specific programming language and IDE (e.g., C, Java, Python) will be provided, or participants can use their preferred setup if allowed.

6. Types of Bugs: The bugs could include syntax errors, logical errors, edge case handling, performance issues, and incorrect outputs.

7. Scoring: Points are awarded based on speed (faster fixes earn more points), accuracy (fixing all bugs), and code clarity (maintaining clean code).
